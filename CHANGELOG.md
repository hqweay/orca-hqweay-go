# Changelog

## 3.0.0

### Major Changes

- 新增大纲导航

### Minor Changes

- 348ca2c: - 新增 `lets-block-nav` 子插件：块结构导航面板
  - 支持显示当前块的子节点列表
  - 支持从编辑器拖拽块到面板作为子节点
  - 支持多选节点（Ctrl/Shift + 点击）
  - 支持批量删除选中节点
  - 支持拖拽排序节点

### Patch Changes

- f7cff0b: - 修复了 Arc Tabs（侧边栏标签页）中标签名称也被强行截断为 20 字符的问题，并为其补充了原生 `title` 悬浮提示，保证在文字过长被省略时也能看清全貌。
- acc809a: - 统一了侧边栏大纲中搜索过滤逻辑和视觉渲染逻辑的口径。过滤时不再生硬地提取底层 `block.text`，而是实时调用底层的纯函数计算出最终将展示在屏幕上的节点标题，以此进行文本匹配。彻底解决了“搜不着看到的，高亮不到搜到的”以及包含属性块匹配断层的问题。
- 96c8dc4: - 修复了侧边栏大纲中长标题被硬编码强制截断为 20 个字符的问题。现在放开了底层数据限制，将其完全交给 CSS `text-overflow: ellipsis` 动态截断，完美解决了截断后导致悬浮提示（Hover Tooltip）内容不全，以及长尾文字无法触发搜索高亮的问题。
- 96c8dc4: - 优化了侧边栏大纲在深层级下的显示体验：为所有大纲节点（包括顶部根节点）增加了原生悬浮提示（Hover Tooltip）。当节点标题因为层级过深或过长被省略号截断时，鼠标悬停即可查看完整的节点文本。
- 0e06116: - 为侧边栏节点添加了右键点击事件（`onContextMenu`）：现在右键点击侧边栏的大纲节点，可以立刻将该节点设为侧边栏的“根节点”（向下聚焦钻取），并与原有的左键点击（在右侧主编辑器打开）逻辑完美区分。
- 0e06116: - 优化了侧边栏节点的右键点击（钻取）体验：现在右键点击侧边栏节点时，不仅会将侧边栏的根节点下钻到该块，同时也会触发编辑器跳转，在右侧主编辑器中同步打开并聚焦该块。
- 0655054: - 为侧边栏大纲搜索功能增加了关键词高亮显示。当处于搜索过滤状态时，所有匹配到搜索词的标题片段都会自动呈现浅黄色的 `<mark>` 高亮背景，极大提升了结果的辨识度。
- 00b0eb8: - 彻底修复了由于笔记图谱中出现循环引用或自引用（`block.parent === block.id`）引发的无限循环死机（崩溃）问题。为 `handleSearch` 和 `resolveAndLoad` 中的层级回溯增加了环路检测和防重入机制，保障侧边栏在极端数据下的绝对稳定性。
- 00b0eb8: - 修复了侧边栏 Filter 输入框可能导致的崩溃问题。
  - 优化了侧边栏 Filter 输入框的 UI：使用了更贴合系统原生的 `CompositionInput` 组件，并将搜索框移至标题下方，以获得更干净的布局。同时 `CompositionInput` 支持中文拼音防抖，在拼音输入过程中不会频繁触发树形重新计算。
- 00b0eb8: - 彻底修复了侧边栏局部搜索时因 `Valtio Proxy` 深度递归代理大体积树对象导致的内存溢出 / 爆栈（闪退）问题。现在采用非响应式的静态内存快照缓存（`searchCache`）来支撑搜索模式下的树渲染，实现搜索过程的零开销与极速响应。
- 00b0eb8: - 修复了侧边栏搜索时触发的 React Error 300 (Rendered fewer hooks than expected) 崩溃问题。这是由于在搜索过滤未命中节点时，过早执行了 `return null` 导致 React Hooks 渲染数量不一致所致，现已将拦截逻辑移至所有 Hooks 声明之后，严格遵守 React Hooks 规范。
- 00b0eb8: - 彻底修复了侧边栏搜索时触发的 React Error 300 崩溃问题。确保提前阻断渲染的 `return null` 位于所有的拖拽 Hook（`useCallback`）之后。
- 4256a64: - 将 Arc Tabs 和 Block Nav 的快捷展开按钮从右上角菜单移到了左上角（紧邻系统的“侧边栏切换”按钮）。这更符合这两个插件作为“全局左侧边栏功能”的直觉定位。
- 3433c61: - 修复了 `lets-arc-tabs` 调用底层布局 API 时由于分裂节点错误导致可能不显示在最左侧或报错的问题：改为递归寻找当前屏幕最左侧的“叶子节点（具体视图面板）”进行向左分裂打开，确保侧边栏永远在全屏的最左侧开启。
- 394cad2: - 修复了 `BlockIcon.tsx` 中的 TypeScript 类型推断错误（由于重新赋值导致的 `ReactNode` 类型丢失）。
- a87ddc8: - 优化了 `lets-block-nav` 的上下文重置逻辑：引入了“家族血脉检测（祖先判定）”算法。当你点击大纲侧边栏跳转到子节点时，侧边栏不再“丢失上下文”发生重置跳转，而是稳如泰山地保持大纲树的稳定，仅更新节点的高亮状态；只有当你彻底离开当前树（比如回退或切换文档）时，侧边栏才会进行合理的重置。
- 67cedf9: - 为 `lets-block-nav` 侧边栏添加了拖拽排序功能：拖拽时鼠标位于节点上方/下方边缘可将其重排为兄弟节点（即相对该节点进行前后排序），位于中心则仍作为子节点落入。
- 00b0eb8: - 为侧边栏大纲增加了“局部全量搜索”功能（双剑合璧方案）。
  - 顶部标题栏右侧新增了一个 Filter 输入框。
  - 当输入过滤词时，会瞬间按需拉取当前树的所有节点并进行过滤，命中的分支会自动展开，未命中的分支和叶子节点会自动隐藏。
  - 清空搜索框后，瞬间恢复到之前极速懒加载的折叠状态。
- 67cedf9: - 修复 `lets-block-nav` 侧边栏节点自身无法被拖拽的问题，为其添加了标准的 `draggable` 属性与 `onDragStart` 事件，现在支持在侧边栏内部直接拖拽节点进行同级排序或层级移动。
- 98010f1: - 史诗级重构：利用 Valtio Proxy 为 `lets-block-nav` 实现了按需懒加载的“无限层级嵌套树状展开/折叠”功能。
  - 在保持初次渲染速度（O(1)级开销）和内存占用极低的前提下，补齐了大纲类笔记跨层级拖拽和全局导航的刚需拼图。
- b642938: - 优化了 `lets-block-nav` 的拖拽回退策略：在没有编辑器时触发回退拉起新编辑器，现在会直接拉起**被拖拽的块**，而不是拖拽的目标块，更符合用户的操作直觉。
- 08ec96d: - 修复了 `lets-block-nav` 展开折叠图标点击后无法立即刷新的 Bug，原因是原有的 `Set` 数据结构不受 Valtio 默认 Proxy 追踪的影响，已将其重构为受支持的 Record（字典）结构。
- c0386be: - 修复了当焦点在侧边栏（而非主编辑器）时，拖拽移动块排序由于缺失编辑器上下文而失败的问题。采用了焦点闪回法（Focus Trick）并在执行前强制将拖拽涉及的块预加载进前端内存。
- a652170: - 为 `lets-block-nav` 侧边栏列表添加了当前聚焦块的选中高亮效果，使用户能更直观地在兄弟节点列表中定位自己当前所处的编辑位置。
- 765a5bf: - 为 `lets-block-nav` 侧边栏的顶部标题栏（当前聚焦块的父节点）也加上了图标和颜色渲染支持，使其视觉效果与列表子节点保持一致。
- df42972: - 优化了 `lets-block-nav` 侧边栏的顶部标题区 UI：现在根节点的标题支持点击跳转聚焦。
  - 在根节点有父节点的情况下，标题左侧会显示一个极简的向上箭头（Up），点击可快速将侧边栏层级向上一层“跳出”，极大提升了空间感和导航效率。
- 394cad2: - 提取并复用了块元信息解析逻辑：在 `libs/utils.ts` 中新增了 `getBlockTitle`, `getBlockIcon`, `getBlockColor` 公共函数，并在 `libs/components` 中抽离了 `<BlockIcon />` 渲染组件。
  - 优化了 `lets-block-nav` 侧边栏：现在完美支持展示节点的自定义图标、日历图标以及自定义颜色了。
  - 重构了 `lets-arc-tabs` 插件，将其原本冗余的内部图标/颜色/标题获取逻辑接入了新的 `libs` 公共方法中。
- bbf224f: - 优化 `lets-block-nav`：现在加载块时，默认加载其兄弟节点（即加载其父节点的子节点），以避免聚焦某个叶子块时侧边栏内容为空。
- 617825f: - 修复 `lets-block-nav` 侧边栏拖拽块到兄弟节点时报错 `No command named core.editor.moveBlock` 的问题，已替换为正确的底层命令 `core.editor.moveBlocks` 并适配了参数格式。
- 6a56bc8: - 修复了 `lets-block-nav` 调用底层布局 API 时由于直接传入容器树根节点导致静默失败无法打开的问题：改为递归寻找当前屏幕最左侧的“叶子节点（具体视图面板）”进行向左分裂打开。
- 6a56bc8: - 修复了 `lets-block-nav` 在开启多个编辑器窗口时打开位置不对的问题：将打开的目标节点由当前的 `activePanel` 修正为整个布局树的 `rootPanel`，确保侧边栏永远只会出现在屏幕的最左侧。
- bbf224f: - 修复 `lets-block-nav` 中追踪活跃编辑器面板时潜在的渲染崩溃问题（空指针异常），导致面板空白的 bug。
- bbf224f: - 修复 `lets-block-nav` 插件中，点击节点时未在正确的主编辑器中打开面板的问题，通过追踪最近活跃的编辑器面板优化导航行为。
- 617825f: - 移除 `lets-block-nav` 侧边栏中无效的 "Expand All" 和 "Collapse All" 按钮及相关逻辑，由于当前采用的是仅加载一层兄弟节点的平面列表模式，树状折叠功能并不适用。
- f60969f: - 移除 `lets-block-nav` 侧边栏中多余的节点选择框（Checkbox）及底部的批量删除功能。作为一个轻量级的大纲导航仪，批量选择和删除操作应在主编辑器内完成，以保持侧边栏 UI 的清爽和功能的专注。
- e0ef6c2: - 修复了 `lets-block-nav` 遗留的 TypeScript 编译报错问题（移除了废弃的 `expandedIds` 依赖项）。
- dc96bc7: - 修复 `lets-block-nav` 侧边栏列表顺序可能与编辑器内不一致的问题。这是因为底层 API `get-blocks` 批量返回块时并不保证顺序，现在强制按照父节点的 `children` 数组顺序对返回数据进行二次重排。
- 3428d47: - 修复了打开多个编辑器面板时，`lets-block-nav` 侧边栏大纲可能展示了上一个失去焦点的编辑器内容的问题（修复了 React useEffect 执行顺序导致的状态滞后）。
- e0ef6c2: - 优化了 `lets-block-nav` 的数据同步机制：现在侧边栏大纲可以实时监听编辑器内的节点新增、删除、排序等操作，并瞬间完成同步刷新。
- f050cc2: - 修复了从编辑器内聚焦/打开块时侧边栏不跟随刷新的问题。现在侧边栏能智能区分“跳转来源”：只有从侧边栏发起的跳转会保持大纲树的稳定，从编辑器发起的聚焦依然会自动重置侧边栏的上下文。
- e0ef6c2: - 修复了在检查面板是否存在时由于 TypeScript 接口定义不匹配引发的类型报错：使用 `isEditorPanel` 递归查询面板树，替代错误的直接字典索引。
- 4592c5e: - 美化 `lets-block-nav` 侧边栏：参考 `lets-arc-tabs` 的 UI 风格，为块节点添加了圆角、更平滑的悬停过渡效果、微小的进入动画、以及更整洁的头部和工具栏边框样式。
- 97ed2b3: - 侧边栏搜索过滤算法优化：现在直接使用底层提供的 `block.text` 字段进行匹配，替代了原先手动解析并拼接富文本 `block.content` 片段的方式，大幅简化了逻辑并提高了文本匹配的准确度。
- 4592c5e: - 修复 `lets-block-nav` 侧边栏宽度可能因布局引擎重算而发生跳变或拉伸的问题，为其添加了与 `lets-arc-tabs` 一致的严格固定宽度逻辑。
- 7f0c086: - 修复了因为 React `useMemo`/`useEffect` 与 Valtio Proxy 对象缓存冲突，导致侧边栏（`arc-tabs` 和 `block-nav`）无法响应“前进/后退”页面内局部刷新的问题。
- 4d4f04b: - 为 `lets-arc-tabs` 的 Pin/Unpin 和 `lets-block-nav` 的拖拽排序加入了更健壮的编辑器焦点回退策略：优先使用静默的“焦点闪回法”，若极端情况下不存在任何激活的编辑器，则回退到 `orca.nav.goTo` 强制拉起编辑器执行命令，保证核心功能始终可用。
- 1ef0384: - 优化 Arc 侧边栏：从侧边栏打开块时，优先使用最近活动的主编辑器面板，而非默认右侧面板
